# Skills Identification Reference

Guidance for extracting and categorizing skills from job descriptions.

## Extracting Skills from a Job Description

### Explicit Skills
Skills directly stated in the JD:
- Listed under "Requirements", "Qualifications", or "Skills"
- Mentioned in bullet points as must-haves or nice-to-haves
- Technical tools, languages, frameworks named by name

### Implied Skills
Skills not stated directly but required by the responsibilities:
- "Collaborate with cross-functional teams" → teamwork, communication, stakeholder management
- "Lead sprint planning" → project management, Agile/Scrum, leadership
- "Debug production issues" → troubleshooting, systems thinking, attention to detail
- "Present findings to leadership" → presentation skills, data storytelling, executive communication
- "Mentor junior engineers" → leadership, teaching, patience, technical communication

### Soft Skills Categories

Categorize identified soft skills by type:

**Communication:**
- Written communication (documentation, reports)
- Verbal communication (presentations, meetings)
- Technical communication (explaining complex topics simply)
- Cross-cultural communication (global teams)

**Leadership:**
- Team leadership (managing, directing)
- Mentorship (teaching, guiding)
- Initiative (self-starting, proactive)
- Decision-making (judgment, prioritization)

**Collaboration:**
- Teamwork (cross-functional, interdisciplinary)
- Stakeholder management (clients, executives)
- Conflict resolution
- Consensus building

**Problem-Solving:**
- Analytical thinking
- Creative problem-solving
- Systems thinking
- Debugging and troubleshooting

**Adaptability:**
- Learning agility (picking up new technologies)
- Flexibility (changing priorities, ambiguity)
- Resilience (handling setbacks)
- Growth mindset

## Prioritizing Skills

Rank skills by importance based on how they appear in the JD:

1. **Critical** — Listed under "Required" or mentioned multiple times
2. **Important** — Listed under "Preferred" or mentioned once prominently
3. **Nice-to-have** — Implied by responsibilities or mentioned in passing
4. **Low priority** — General skills that any candidate would have

## Demonstrating Skills Through Academic Experience

For students with limited industry experience, map skills to academic contexts:

| Skill | Academic Demonstration |
|---|---|
| Project management | Led capstone project with X team members over Y months |
| Technical communication | Presented research at X conference / Wrote technical reports |
| Collaboration | Worked in teams of X on group projects / Cross-lab collaboration |
| Problem-solving | Designed novel approach to X research problem |
| Data-driven decisions | Used statistical analysis to validate hypothesis |
| Leadership | Served as TA, led study groups, organized events |
| Time management | Balanced full course load with research and part-time work |
| Adaptability | Pivoted research direction when initial approach failed |
