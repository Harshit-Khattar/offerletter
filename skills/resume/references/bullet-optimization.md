# Bullet Point Optimization Reference

Rules for converting task-oriented bullets to achievement statements and removing redundancy.

## Tasks vs Accomplishments

### How to Identify a Task Bullet
Task bullets describe what someone did — the activity, not the result:
- "Responsible for managing the database"
- "Worked on the frontend of the application"
- "Assisted with data analysis"
- "Participated in code reviews"
- "Helped with testing"

### How to Identify an Accomplishment Bullet
Accomplishment bullets describe the impact — the outcome, not just the activity:
- "Optimized database queries, reducing average response time by 45%"
- "Built responsive frontend components serving 10K+ daily active users"
- "Analyzed customer churn data, identifying 3 key retention drivers that informed a $2M strategy shift"

### Conversion Framework

For each task bullet, ask these questions:
1. **What did you actually do?** (the action)
2. **How did you do it?** (the method/tools)
3. **What was the result?** (the impact)
4. **Can you quantify it?** (the numbers)

Formula: **[Action Verb] + [What You Did] + [Using What] + [Result/Impact]**

### Quantification Strategies

When the user says "I don't have numbers", help them estimate:

| Type | Questions to Ask |
|---|---|
| Scale | How many users/records/transactions? How large was the dataset? |
| Speed | How much faster? What was before vs after? |
| Money | What was the budget? Revenue impact? Cost savings? |
| Time | How many hours saved? How much faster was delivery? |
| Quality | Error rate reduction? Accuracy improvement? |
| Team | How many people? How many mentored? |
| Frequency | How often? Daily/weekly/monthly volume? |

If exact numbers aren't available, use ranges or approximations:
- "50K+" instead of exact count
- "~30% improvement" for estimates
- "Multiple" or "Several" as a last resort (weakest)

## Action Verb List

### Technical/Engineering
Built, Developed, Designed, Implemented, Architected, Engineered, Deployed, Optimized, Automated, Refactored, Migrated, Integrated, Scaled, Debugged, Configured

### Research/Analysis
Analyzed, Researched, Investigated, Evaluated, Assessed, Modeled, Validated, Tested, Benchmarked, Measured, Quantified, Discovered, Identified

### Leadership/Management
Led, Managed, Directed, Coordinated, Mentored, Supervised, Organized, Facilitated, Initiated, Drove, Championed, Spearheaded

### Communication
Presented, Documented, Published, Reported, Communicated, Demonstrated, Explained, Authored, Translated (technical to non-technical)

### Improvement
Improved, Enhanced, Streamlined, Reduced, Increased, Accelerated, Simplified, Consolidated, Modernized, Upgraded

## Redundancy Detection

### Signs of Redundancy
- Multiple bullets starting with the same action verb
- Several bullets demonstrating the same skill (e.g., three bullets all showing "Python experience")
- Overlapping descriptions across different jobs/projects
- Generic bullets that could apply to anyone ("Collaborated with team members")

### How to Fix Redundancy
1. **Identify the unique angle** of each bullet — what specific skill or achievement does it showcase?
2. **Merge overlapping bullets** — combine two weak bullets into one strong one
3. **Diversify action verbs** — no verb should appear more than twice on the resume
4. **Ensure each bullet adds unique value** — if removing a bullet wouldn't lose any information, remove it
5. **Prioritize by relevance** — keep the bullets most relevant to the target role

### Before/After Redundancy Fix

Before (redundant — all show "teamwork"):
- "Collaborated with team to develop new features"
- "Worked with cross-functional team on product launch"
- "Partnered with designers to improve UX"

After (each shows a different strength):
- "Led a 4-person team to deliver a product feature 2 weeks ahead of schedule" (leadership)
- "Translated designer mockups into responsive React components, reducing design-to-code cycle by 30%" (technical skill + speed)
- "Coordinated cross-functional launch across engineering, design, and marketing for a product serving 50K users" (project management + scale)
