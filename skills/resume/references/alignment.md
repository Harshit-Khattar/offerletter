# Resume-JD Alignment Reference

Detailed guidance for tailoring a resume to match a specific job description.

## Analysis Approach

### Step 1: Deconstruct the Job Description

Break the JD into these categories:
- **Must-have technical skills** — explicitly listed as "required"
- **Nice-to-have technical skills** — listed as "preferred" or "bonus"
- **Soft skills** — communication, leadership, collaboration, etc.
- **Domain knowledge** — industry-specific terms, methodologies, frameworks
- **Responsibilities** — what the role actually does day-to-day
- **Company values** — clues about culture from the JD language

### Step 2: Map Resume to JD

For each JD requirement, find the closest match in the resume:
- Direct match: skill or experience explicitly mentioned
- Indirect match: related skill or transferable experience
- Gap: no match found

### Step 3: Section-by-Section Tailoring

**Skills Section:**
- Reorder skills to prioritize those mentioned in the JD
- Add relevant skills the user has but didn't list
- Remove or de-emphasize skills irrelevant to this role
- Mirror the JD's terminology (e.g., if JD says "React.js", don't write "React")

**Experience Bullets:**
- Lead with the most relevant experience for this role
- Rewrite bullets to emphasize aspects that align with JD responsibilities
- Add quantified impact where possible (numbers, percentages, scale)
- Use action verbs that match the JD's language

**Projects Section:**
- Prioritize projects that demonstrate skills listed in the JD
- Add technical details that match JD requirements
- Emphasize outcomes and impact over process

**Summary/Objective (if present):**
- Tailor to mention the specific role and company
- Highlight 2-3 key qualifications that directly match the JD
- Keep to 2-3 sentences maximum

## Keyword Integration Strategy

- Identify the top 15-20 keywords from the JD
- Integrate naturally — never keyword-stuff
- Aim for each important keyword to appear at least once
- Use exact phrasing from the JD when possible (ATS systems match exact strings)
- Place keywords in context, not in isolation

## Skills Translation (Academic to Industry)

When the user has academic experience, translate it:

| Academic Language | Industry Language |
|---|---|
| Conducted research on... | Developed and implemented... |
| Published findings in... | Delivered technical documentation for... |
| Assisted professor with... | Collaborated with senior team on... |
| Coursework in... | Trained in... / Proficient in... |
| Thesis on... | Led end-to-end project on... |
| Lab work involving... | Hands-on experience with... |
| Teaching assistant for... | Mentored and trained team of X students in... |

## Technical Skills Optimization

When reorganizing the technical skills section:

1. **Group by category**: Programming Languages, Frameworks, Tools, Databases, Cloud, etc.
2. **Order by relevance**: Most relevant to the JD comes first within each category
3. **Remove outdated skills**: Unless the JD specifically asks for them
4. **Match proficiency to claims**: Don't list skills the user can't speak to in an interview
5. **Include version numbers only when relevant**: e.g., "Python 3" only if the JD specifies

## Project Description Enhancement

For each project bullet:

1. **Start with a strong action verb** — Built, Developed, Designed, Implemented, Optimized
2. **Include the technical stack** — What technologies were used
3. **Quantify the outcome** — Users served, performance improvement, data processed
4. **Connect to business value** — Why did this matter?

### Before/After Examples

Before: "Worked on a machine learning project for sentiment analysis"
After: "Built a sentiment analysis pipeline using Python and BERT, achieving 94% accuracy on 50K+ customer reviews, enabling automated feedback categorization"

Before: "Created a web application for the school"
After: "Developed a full-stack web application (React, Node.js, PostgreSQL) serving 2,000+ students, reducing course registration time by 40%"

Before: "Did data analysis for research"
After: "Analyzed 100K+ clinical trial records using pandas and scikit-learn, identifying 3 statistically significant biomarkers that informed the research team's next phase"
