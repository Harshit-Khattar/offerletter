# Role Mapping Reference

## Process

1. Extract from resume: technical skills, domain knowledge, experience type, education, interests
2. Map to role families:

| Background | Target Roles |
|---|---|
| Coding + web frameworks | SWE, Full-Stack, Frontend/Backend |
| Data + stats + viz | Data Analyst, BI Analyst, Analytics Engineer |
| ML/AI + Python + models | Data Scientist, ML Engineer, AI Engineer |
| Systems + cloud | DevOps, SRE, Platform Engineer |
| Research + papers | Research Scientist/Engineer |
| NLP + LLMs | NLP Engineer, AI Engineer |

3. Differentiate similar roles (DS vs ML Eng vs AI Eng — analysis vs production vs application)
4. Score fit: technical match (30%), experience (25%), interest (20%), market demand (15%), growth trajectory (10%)
5. Fit levels: Strong 80%+ / Good 60-79% / Stretch 40-59%

## Gap Bridging

| Gap | How to Bridge | Timeline |
|---|---|---|
| Missing language | Course + project | 2-4 weeks |
| Missing framework | Tutorial + project | 1-3 weeks |
| Missing domain | Course + blog + project | 1-3 months |
| Missing certification | Study + exam | 1-6 months |

## Entry-Level Title Variants

SWE = SDE = Software Developer = Application Developer
Data Scientist = Junior DS = Associate DS
ML Engineer = ML SWE = AI/ML Engineer

## Job Search Strategy

High-demand roles (SWE, Data Analyst): apply broadly, differentiate through projects
Specialized roles (ML Eng, NLP Eng): targeted applications, portfolio matters more
Competitive roles (DS, Research): consider adjacent roles as stepping stones
