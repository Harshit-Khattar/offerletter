# Skills-to-Roles Mapping Reference

Logic for identifying target job titles from a student's background and mapping skills to roles.

## How to Identify Target Roles

### Step 1: Extract the Student's Profile

From their resume and conversation, identify:
- **Technical skills** — languages, frameworks, tools, platforms
- **Domain knowledge** — ML, NLP, data engineering, web dev, security, etc.
- **Experience type** — research, internships, projects, teaching, open source
- **Education** — degree, specialization, relevant coursework
- **Interests** — what they enjoy (from self-assessment if available)

### Step 2: Map to Role Families

| If They Have... | Consider These Roles |
|---|---|
| Strong coding + web frameworks | Software Engineer, Full-Stack Developer, Frontend/Backend Engineer |
| Data analysis + statistics + visualization | Data Analyst, Business Intelligence Analyst, Analytics Engineer |
| ML/AI + Python + model building | Data Scientist, ML Engineer, AI Engineer, Applied Scientist |
| Systems + infrastructure + cloud | DevOps Engineer, SRE, Platform Engineer, Cloud Engineer |
| Research + publications + experiments | Research Scientist, Research Engineer, Applied Researcher |
| Product sense + technical + communication | Technical Product Manager, Solutions Engineer, Developer Advocate |
| Security + networking + systems | Security Engineer, Application Security, SOC Analyst |
| NLP + LLMs + text processing | NLP Engineer, AI Engineer, Conversational AI Engineer |
| Computer vision + image processing | CV Engineer, Perception Engineer, Robotics Engineer |
| Data pipelines + ETL + databases | Data Engineer, Analytics Engineer, Database Engineer |

### Step 3: Differentiate Similar Roles

Students often confuse similar-sounding roles. Clarify:

**Data Scientist vs ML Engineer vs AI Engineer:**
- **Data Scientist**: Analysis-focused. Explores data, builds models, communicates insights. More Jupyter notebooks, less production code
- **ML Engineer**: Production-focused. Takes models from notebook to production. Builds pipelines, monitors performance. More software engineering
- **AI Engineer**: Application-focused. Integrates AI capabilities (LLMs, APIs) into products. More product engineering, less research

**Software Engineer vs Software Developer vs Full-Stack Developer:**
- These are functionally the same role at most companies. Title varies by company convention
- "Full-Stack" means they expect both frontend and backend skills
- At big tech, "SWE" is the standard title regardless of specialization

**Data Analyst vs Data Scientist:**
- **Data Analyst**: Descriptive analytics, dashboards, SQL-heavy, business-facing, shorter projects
- **Data Scientist**: Predictive/prescriptive analytics, model building, Python/R-heavy, more autonomy, longer projects

**DevOps vs SRE vs Platform Engineer:**
- **DevOps**: CI/CD, deployment automation, developer tooling. Enables other engineers
- **SRE**: Reliability, monitoring, incident response, SLOs. Keeps things running
- **Platform**: Internal tools, infrastructure abstractions, developer experience. Builds the platform others build on

## Scoring Role Fit

For each potential role, score against the student's profile:

| Factor | Weight | Score (1-5) |
|--------|--------|-------------|
| Technical skill match | 30% | How many required skills do they already have? |
| Experience alignment | 25% | Do their projects/internships relate? |
| Interest alignment | 20% | Do they WANT to do this type of work? |
| Market demand | 15% | How many openings exist at their level? |
| Growth trajectory | 10% | Does this role lead where they want to go? |

**Fit levels:**
- **Strong fit (80%+)**: Can apply now with current skills
- **Good fit (60-79%)**: Can apply with minor resume tailoring and 1-2 skill additions
- **Stretch (40-59%)**: Possible but needs significant development in key areas
- **Not recommended (<40%)**: Major skill gaps — suggest alternative roles

## Skills Gap Analysis

For each gap between the student's profile and target role:

| Gap Type | How to Bridge | Timeline |
|---|---|---|
| Missing programming language | Online course + project | 2-4 weeks |
| Missing framework/tool | Tutorial + side project | 1-3 weeks |
| Missing domain knowledge | Course + blog posts + project | 1-3 months |
| Missing experience type | Personal project or volunteering | 1-3 months |
| Missing certification | Study + exam | 1-6 months |
| Missing degree/education | Not bridgeable short-term — focus on demonstrating skills through projects |

## Entry-Level Title Variants

Companies use different titles for the same level. Help students recognize these:

| Standard Title | Also Called |
|---|---|
| Software Engineer | SDE, Software Developer, Application Developer, Associate Engineer |
| Data Scientist | Junior Data Scientist, Associate Data Scientist, Data Science Analyst |
| ML Engineer | Machine Learning Software Engineer, AI/ML Engineer |
| Data Analyst | Business Analyst, Analytics Associate, Insights Analyst |
| Data Engineer | ETL Developer, Analytics Engineer (sometimes), Pipeline Engineer |
| DevOps Engineer | Infrastructure Engineer, Build Engineer, Release Engineer |
| Product Manager | Associate PM, Technical PM, Program Manager (sometimes) |

## Job Search Strategy by Role Type

**High-demand roles (SWE, Data Analyst):**
- Many openings → be selective about company/team
- Differentiate through projects and culture fit
- Apply broadly but customize per company

**Specialized roles (ML Engineer, NLP Engineer):**
- Fewer openings → targeted applications are more important
- Portfolio/GitHub matters more
- Conference papers or blog posts can set you apart

**Competitive roles (Data Scientist, Research Scientist):**
- Very competitive at top companies → have backup options
- Research experience or publications help significantly
- Consider adjacent roles as stepping stones (Data Analyst → Data Scientist)
